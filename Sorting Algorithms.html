Sorting Algorithms

In Java, we have a plethora of different sorting algorithms that work differently but achieve the same result, a sorted array of data. Each sorting algorithm has its' best uses. Listed below are the major sorting algorithms you will find on the AP Exam and their implementation.

  1. Insertion Sort
    • Insertion Sort repeatedly compares the next number in the data to the numbers already compared and inserts the number where it belongs.
    • Stable: relative order of elements with equal keys won't change
    • Worst-case performance: О(n^2) comparisons and swaps
    • Best-case performance: O(n) comparisons, O(1) swaps
    • Average performance: О(n^2) comparisons and swaps
    • Worst-case space complexity: О(n) total, O(1) auxiliary
  2. Selection Sort
    • Selection Sort repeatedly finds the smallest number in the list and swaps it with the first index of the list. Through every iteation, the list taht the algorithm sorts through gets smaller each time.
    • Worst-case performance: О(n2) comparisons, О(n) swaps
    • Best-case performance: О(n2) comparisons, О(n) swaps
    • Average performance: О(n2) comparisons, О(n) swaps
    • Worst-case space complexity: O(1) auxiliary
  3. Merge Sort
    • Merge Sort uses a divide and conquer strategy when sorting data in a list.
    • Merge Sort continuosly divides the data until each entry is in it's own list. The algorithm then joins these individual lists, and sorts them until it becomes one sorted list.
    • Worst-case performance: O(n log n)
    • Best-case performance: O(n log n) typical, O(n) natural variant
    • Average performance: O(n log n)
    • Worst-case space complexity: О(n) total with O(n) auxiliary, O(1)
