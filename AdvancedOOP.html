Advanced Object-Oriented Program

Although we discussed the basic concepts of object-oriented programming in the previous tutorial, OOP is a bit more complex than previously explained. In this tutorial we will further discuss more advanced concepts such as Polymorphism, abstract classes and Interfaces. Here is a brief description of each new concept below:

  1. Polymorphism:
    • Polymorphism is the concept that child classes may sometimes want to act different than their parent classes.
    • Java allows this by allowing child classes to override methods from the parent class and implement them in a way that is unique to the child class.
    • In the example below, we see that all objects of the People class sleep for 7 hours. But objects of the Baby class, a descendant of the People class, sleeps for 16 hours instead of 7, while objects of the Child class sleep for 9 hours rather than 7.
    • If an object uses a reference variable of the parent class, but was constructed using the child constructor, that object can be downcast in order to access methods unique to the child class.
    • The computer decides to use the overridden method is during compile-time and is this process is called static binding.

  2. The abstract Class
    • Abstract classes must be extended.
    • If a class extends an abstract class, all abstract methods must be implemented
    • You can create a reference variable of an abstract class type, but you cannot instantiate abstract classes with new objects.
    • Abstract classes must have the keyword abstract.

  3. The Interface
    • Interfaces cannot have instance variables, constructors or methods with implementation.
    • An Interface is an agreement with the class that implements it to use all the methods defined within the interface.
    • An Interface uses the keyword implements.
    • A class can implement more than one interface .
    • You can create a reference variable of an interface type, but you cannot instantiate interfaces with new objects.
