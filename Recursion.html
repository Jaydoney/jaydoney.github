Recursion

Recursion is the process of repeating instructions by calling the same method within itself. Recursion takes a problem and breaks it down into smaller pieces. Recursive methods are often equiped with a base case to stop the recursive method from calling itself once a condition is met.

Key Facts

  • Recursion works better than iteration when the problem can be shrunken into a single, final simple problem
  • Developers must be careful and ensure their recursive method has a base case, otherwise they run the risk of calling themselves an infinite amount of times until a StackOverflowError is thrown.
  • If a recursive method is called an excssive amount of times, it could take up a lot of space in the stack as everytime the method calls itself, the result must be stored in the stack.

    Analysis: In this problem, we are looking to create a recursive method that takes in a decimal integer as a paramater and returns a binary integer. This problem can be easily solved using recursion as a binary is comprised of base 2 to the power of n.

    Explanation: To begin solving this problem. We must begin with what we know:

  • Decimal numbers are comprised of base 10's to the power of n
  • Binary numbers are comprised of base 2's to the power of n
    • This means in order to conver a decimal number to a binary number, we must repeatedly divide the decimal number by 2 and track the remainders. The remainders will represent the Base 2's and if we add them all together we can get the Binary number.
    I started solving this problem by creating a base case of if the decimal number is 0, return 0. This is because 0 in decimal is also 0 in binary. I then created the recursive call by asking the method to return the remainder of the decimal divided by 2 + the decimal to binary conversion of the decimal number divided by 2 * 10. Multiplying by 10 ensures that each place will have one digit, either a 1 or a 0.
